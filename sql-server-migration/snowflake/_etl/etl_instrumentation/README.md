# etl_instrumentation

This directory contains instrumentation resources for tracking row count metrics during ETL execution.

## Contents

### configuration/

Utilities for instrumenting code converted from SSIS to Snowflake technologies (dbt, task graphs, etc).

#### procedures/

- `collect_table_metrics.sql` — Collects row counts for tables related to a task and logs them to `TABLE_METRICS_LOG`
- `ssis_read_baseline_data.sql` — Loads baseline data from instrumented SSIS execution logs into `BASELINE_TABLE_METRICS_LOG`
- `ssis_read_control_flow_config_from_stage.sql` — Parses XML configuration files from a stage and populates `CONTROL_FLOW_TABLE_USAGE`

#### scripts/

- `instrument_generated_dbt_project.py` — Adds post-hook macros to `dbt_project.yml` for metrics tracking
- `instrument_generated_task.py` — Inserts `COLLECT_TABLE_METRICS` calls into generated Snowflake SQL

#### tables/

- `ssis_baseline_metrics_log_table.sql` — Stores baseline metrics from instrumented SSIS packages
- `ssis_task_table_usage.sql` — Tracks which tables are affected by each SSIS task
- `table_metrics_log.sql` — Stores metrics from converted code execution

### source/

Instrumented version of the original SSIS packages. These packages have been modified to track row count metrics after the execution of each step.

> **Note:** Files with the `_control_flow_configuration.xml` suffix (e.g., `TestDemoPackage_control_flow_configuration.xml`) are generated by SnowConvert. They contain information about which tables are used by each control flow task, derived from SnowConvert's internal lineage analysis.

## Instructions

### Enabling Instrumentation of converted code

1. Deploy all `.sql` files from `Configuration/procedures/` and `Configuration/tables/`
2. Load all `*_control_flow_configuration.xml` files into the `CONTROL_FLOW_TABLE_USAGE` table using the `READ_CONTROL_FLOW_CONFIG_FROM_STAGE` procedure
3. Enable instrumentation for tasks using `Configuration/scripts/instrument_generated_task.py`
4. Enable instrumentation for dbt projects using `Configuration/scripts/instrument_generated_dbt_project.py`
5. Redeploy the modified tasks and dbt projects
6. Execute the tasks to be monitored
7. Check results in `PUBLIC.TABLE_METRICS_LOG`

### Using the Instrumented SSIS Code

The `source/` directory contains instrumented versions of the DTSX projects, modified to register row count data at different steps of execution.

1. Copy these files to a machine where the original packages can execute. This requires:
    - SQL Server Data Tools with Visual Studio
    - Access to data sources compatible with the original code (databases with test or synthetic data)

2. Open the instrumented package and verify the `SnowConvertLogsDirectory` variable points to:
    - The directory containing the `<package_name>_control_flow_configuration.xml` file
    - This directory will also store measurement data

3. Execute the package in Visual Studio

4. Validate the execution results. If errors occur, see the Troubleshooting section

5. Collect results:
    - Control flow metrics: `<package_name>_control_flow_results.csv`
    - Data flow metrics (SSIS execution log): `<package_name>_data_flow_log.csv`

## Troubleshooting

**Connection authentication issues**

If measuring a table requires a connection that uses authentication other than `Integrated Security`, add an element to override the connection string in the configuration file:

```xml
<ConnectionString Id="CONNECTION MANAGER ID">NEW CONNECTION STRING TO USE</ConnectionString>
```
